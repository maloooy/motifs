# Motifs

Приложение на **C++17** для поиска ДНК-мотивов в последовательностях формата FASTA с использованием **OpenMP** и **MPI**.

Проект позволяет выполнять поиск большого количества мотивов в наборе последовательностей с возможностью параллельного исполнения как на одном компьютере, так и на кластере.

---

# Возможности

* Поиск мотивов в последовательностях FASTA.
* Поддержка пользовательского списка мотивов.
* Ограничение длины анализируемого мотива.
* Параллельное выполнение с использованием OpenMP.
* Распределённые вычисления с использованием MPI.

---

# Требования

Для сборки необходимы:

* C++17
* GCC с поддержкой OpenMP
* Make

# Использование

Общий формат запуска

```bash
./main \
    --fasta <FASTA_FILE> \
    --motifs <MOTIF_FILE> \
    --threads <THREADS> \
    [--motif-len <LENGTH>]
```

---

# Параметры

| Параметр      | Описание                          |
| ------------- | --------------------------------- |
| `--fasta`     | файл с последовательностями FASTA |
| `--motifs`    | файл со списком мотивов           |
| `--motif-len` | длина анализируемой части мотива  |
| `--threads`   | количество потоков OpenMP         |
| `--help`      | вывести справку                   |

Параметры `--fasta`, `--motifs` и `--threads` являются обязательными.
Параметр `motif-len` по умолчанию равен 8.

---

# Пример запуска

Используя данные из репозитория

```bash
./main \
    --fasta data_5k/FoxA2_5000.fst \
    --motifs data_5k/FoxA2_major_30.mot \
    --motif-len 8 \
    --threads 8
```

---

# Запуск с MPI

После сборки

```bash
make mpi
```

приложение можно запускать, например, на четырёх процессах

```bash
mpirun -np 4 ./main \
    --fasta data_5k/FoxA2_5000.fst \
    --motifs data_5k/FoxA2_major_30.mot \
    --motif-len 8 \
    --threads 4
```

Параметр `-np` задаёт количество MPI-процессов, между которыми распределяется обработка данных.
Количество процессов MPI и количество потоков OpenMP задаются независимо.

# Входные данные

## FASTA

Файл должен содержать последовательности ДНК в стандартном формате FASTA.

Пример

```text
>sequence_1
ATCGTTAGCGAT...

>sequence_2
CGATCGATCGTA...
```

---

## Файл мотивов

Из каждой строки считывается первое слово. Если его длина превышает значение `motif-len`, используется только первые `motif-len` символов.

Пример

```text
ATCGTGCA
CGATAGCT
TTCGATGA
```

---

# Выходные данные

Результаты выводятся в формате TSV.

Для каждого мотива отображается информация о количестве найденных совпадений в обработанных последовательностях.

```text
motif      matches  sequences   frequency
TRTWKACH   1832     5000        0.3664
RTTKACHY   1452     5000        0.2904
TMAAYANS   1491     5000        0.2982
...
```

---

# Алгоритм работы

1. Загружаются последовательности FASTA.
2. Загружается список мотивов.
3. Проверяется корректность входных данных.
4. Выполняется предварительная подготовка мотивов.
5. Последовательности распределяются между потоками OpenMP и (при использовании MPI) между процессами.
6. Выполняется поиск мотивов.
7. Результаты собираются и выводятся в формате TSV.

---

# Ошибки

Приложение завершит работу с сообщением об ошибке в следующих случаях:

* отсутствует файл FASTA;
* отсутствует файл мотивов;
* не указано число потоков;
* длина мотива меньше либо равна нулю;
* FASTA не содержит последовательностей;
* файл мотивов пуст;
* ни один мотив не имеет достаточной длины.

---

# Производительность

Проект поддерживает два уровня параллелизма:

* **OpenMP** — многопоточное выполнение на одном компьютере;
* **MPI** — распределённые вычисления на нескольких процессах или узлах.

При совместном использовании MPI и OpenMP возможно эффективно обрабатывать большие наборы последовательностей.
